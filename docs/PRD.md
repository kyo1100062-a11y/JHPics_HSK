  * 현재 페이지 삭제 버튼 (2페이지 이상일 때만 활성화)
- 메인 캔버스 영역: A4Canvas 표시 영역 (좌측 넓은 영역)
- 우측 사이드바: 출력 옵션 (우측 좁은 영역)

A4Canvas: 실제 A4 비율 유지

▷ 전체 A4Canvas 구조

(1) A4 전체 크기: 210mm × 297mm

(2) 캔버스 마진(Canvas Margin): A4 경계와 OuterFrame 사이에 상·하·좌·우 각각 20mm

(3) OuterFrame (외곽 프레임): 2px, solid black
- 캔버스 마진(20mm) 안쪽에 배치

- OuterFrame 내부에는 8mm 내부 여백(padding) 존재

(4) OuterFrame 내부 영역:
- Metadata Area (메타데이터 영역)
- Image Area (이미지 슬롯 영역)

OuterFrame → Metadata → ImageArea 순으로 상단에서 하단으로 배치된다.

6.1.1 OuterFrame (외곽 프레임)

▷ 배치 및 크기
- OuterFrame은 A4 전체 영역 내부에 배치되며, A4 경계와 OuterFrame 사이에는 상·하·좌·우 각각 20mm의 캔버스 마진(Canvas Margin)을 둔다.
- OuterFrame: 2px, solid black
- OuterFrame 내부 여백: 8mm (상·하·좌·우)
- 캔버스 배경: 흰색

▷ 크기 고정 규칙
- 이미지 로드 여부와 무관하게 OuterFrame 및 캔버스 마진의 크기는 절대 변경되지 않는다.
- CSS aspect-ratio 또는 JavaScript 기반 크기 강제로 A4 비율(210mm × 297mm)을 유지한다.
- 메타데이터 영역과 이미지 영역은 OuterFrame 내부 여백(8mm) 안에서만 배치된다.

6.1.2 Metadata Area (메타데이터 영역)
- 전체 A4Canvas 상단에 배치
- 텍스트 정렬: 좌측 정렬
- 제목의 글씨 크기는 다른 항목보다 조금 더 큰 크기 사용
- 메타데이터 영역과 이미지 영역의 간격은 6mm로 고정한다.
- 모든 메타데이터 텍스트 색상: 매우 짙은 회색 (#333333)

▷ 메타데이터 영역 높이 규칙
- 높이는 폰트 크기 기준으로 계산된다.
- 최소 높이: 1줄(제목+사업명) 기준 높이 (폰트 크기 + 줄 간격)
- 최대 높이: 3줄(제목+사업명, 보조사업자, 담당자)이 모두 보이는 높이로 제한
- 전체 A4Canvas 높이 기준으로 약 10~20% 범위 안에서 동적으로 조정
- 줄 간 간격: 3mm 정도로 유지



▷ 구성: 3줄 구성(최대)
- 1줄: 제목 + [사업명 : 사업명값] 형식으로 표시 (한 줄에 표기, 엑셀 텍스트 셀맞춤과 동일)
- 2줄: 보조사업자 : 보조사업자값 (수기입력)
- 3줄: 담당자: 담당자값 (수기입력)
- "사업명", "보조사업자"와 콜론 사이에는 공백 한 칸 포함
- 보조사업자와 담당자 줄 간격은 다른 줄보다 30% 작게 설정

▷ 줄 표현 규칙
- 보조사업자/담당자 입력이 비어있으면 해당 줄은 제거
- 즉, 메타데이터 영역 전체는 동적(가변형)
- 예: 3줄 → 2줄 또는 1줄로 줄어듦

6.1.3 Image Area (이미지 슬롯 영역)
- 이미지 영역 상단: 메타데이터 영역 하단과 6mm 간격
- 이미지 영역 좌/우/하단: OuterFrame 내부 여백과 동일하게 각 8mm
- 슬롯 간 간격: 2mm


▷ 슬롯 배치 규칙
- 템플릿(slot 2/4/6)은 고정 레이아웃
- 커스텀(slot 1~16)은 이미지 영역 내 그리드/드래그/리사이즈 규칙 적용
- 슬롯은 메타영역·아웃프레임과 겹칠 수 없음

페이지 추가: 버튼 클릭 시 A4 페이지 추가

6.3 기본 템플릿 슬롯 규칙 (2·4·6컷)

템플릿 2·4·6은 고정 슬롯 수를 가진다.

슬롯은 이미지 영역 내부에서 자동 정렬되며,
슬롯 간 겹침 없음, 메타데이터 영역과 겹침 없음, OuterFrame과 겹침 없음을 기본 규칙으로 한다.

슬롯 크기는 이미지 영역 내에서 가능한 범위에서 최대 크기로 자동 조정된다.

슬롯 간 여백은 모든 슬롯에서 동일한 간격을 유지한다.

방향(세로형/가로형)에 따라 슬롯 배열이 변경된다:
- 세로형(포트레이트) → 2컷(1×2), 4컷(2×2), 6컷(2×3)
- 가로형(랜드스케이프) → 2컷(2×1), 4컷(2×2 동일), 6컷(3×2)

위 템플릿은 고정 템플릿이며 커스텀 레이아웃과는 별개로 동작한다.

생성된 슬롯은 다음 기능 제공:

크기 조절 핸들바(사이즈 조정)

커스텀 슬롯의 배치는 사용자가 자유롭게 드래그·리사이즈할 수 있으나, 드래그 또는 리사이즈가 종료되는 시점에 슬롯의 위치와 크기는 공통 그리드(Layout Grid)에 자동 스냅되어 정렬되도록 한다. 즉, 사용자의 자유 배치는 허용하되 최종 결과는 그리드 기준으로 정돈되는 형태로 동작한다.

6.4.1 Custom Template 제약

슬롯 최대 개수: 16개

슬롯 제한 규칙(커스텀 템플릿)
- 슬롯 최대 크기: 이미지 영역(메타데이터 영역 제외) 내부에서만 확장 가능
- 슬롯 겹침: 허용하지 않음
- 슬롯 위치: A4 전체가 아닌 '이미지 슬롯 전용 영역' 내부에서만 생성·이동 가능
- 슬롯 간 겹침(Overlap) 금지, 충돌 시 이동/확장 불가
- 슬롯 최소 크기: 300px × 300px 이상 (이미지 영역 기준)
- 슬롯 최대 크기: 이미지 영역 내 허용 범위까지 확장
- 기본적으로 자동 적용 슬롯은 균등한 크기를 가진다
- 설명 텍스트가 있을 경우에도 이미지 영역의 최소 크기는 300px × 300px을 유지하며, 텍스트 영역은 자동 확장된다.


수동 조정(핸들바 사용)
- **핸들바(우하단 Resize Handle)**를 이용해 개별 슬롯의 크기 조정 가능
- 조정 시 적용 규칙:
  - 최소 크기 이하로 축소 불가
  - 이미지 영역 밖으로 확장 불가
  - 다른 슬롯과 충돌 시 조정 불가
  - 드래그 또는 리사이즈 종료 시점에 공통 그리드 기반 자동 스냅(Snap-to-grid) 기능이 적용된다.
    * A4 이미지 영역을 rows × cols 그리드로 나눈 뒤, 슬롯의 x, y, width, height가 이 그리드 라인에 맞춰 스냅되도록 한다.
    * 드래그/리사이즈가 끝날 때는 가까운 그리드 라인으로 스냅되어 슬롯 간 줄 맞춤이 자연스럽게 유지된다.
    * 예: 3×4, 4×3, 4×4 등 공통 그리드에 정렬된다.


6개 슬롯 규칙 (중요)
- 세로형: 2열 × 3행
- 가로형: 3열 × 2행

12개 슬롯 규칙 (중요)
- 세로형: 3열 × 4행
- 가로형: 4열 × 3행

▷ 이미지 표시 모드
- 기본값: "전체 표시 (왜곡 허용)" - object-fit: fill
  * 이미지 업로드 시 기본적으로 적용됨
  * 이미지가 슬롯 크기에 맞춰 늘어나거나 줄어들어 전체 이미지가 표시됨
  * 이미지의 가로·세로 비율이 왜곡될 수 있으나, 이미지 누락 없이 전체가 표시됨
- 선택 옵션: "비율 유지 (일부 잘림 허용)" - object-fit: cover
  * 업로드된 이미지에 마우스를 올리면 "비율유지" 버튼이 표시됨
  * "비율유지" 버튼 클릭 시 해당 슬롯의 이미지가 "비율 유지" 모드로 전환됨
  * 이미지의 원본 비율을 유지하면서 슬롯을 꽉 채움
  * 이미지의 일부가 잘릴 수 있으나, 왜곡 없이 표시됨
- 각 슬롯별로 독립적으로 표시 모드를 설정할 수 있음
- 새 이미지를 업로드하면 기본값("전체 표시")으로 리셋됨

▷ 레이아웃 고정 규칙
- 이미지가 슬롯에 로드되기 전에 A4Canvas와 OuterFrame의 크기를 고정한다.
- 이미지 로드 시에도 아웃프레임 및 캔버스 마진 크기는 변경되지 않으며, 이미지만 슬롯 내부에서 조정된다.
- 기본적으로 object-fit: fill을 사용하여 슬롯 크기에 맞춘다 (전체 표시, 왜곡 허용).
- 사용자가 "비율유지" 버튼을 클릭하면 object-fit: cover로 전환된다 (비율 유지, 일부 잘림 허용).
- 다양한 비율의 원본 이미지(가로가 매우 긴 사진, 세로가 매우 긴 사진 등)가 동일한 페이지의 각 슬롯에 업로드되어도 OuterFrame 크기는 절대 변경되지 않는다.

8.2.1 빈 슬롯 디자인
- 점선 테두리 (dashed border), 점선색상: 매우 짙은 회색
- 중앙에 큰 "+" 아이콘
- "+" 아이콘 하단에 "사진 추가 또는 드래그하여 업로드" 텍스트
- 배경: 연한 회색 또는 흰색

8.2.2 이미지 업로드 후 슬롯 디자인
- 이미지가 업로드된 슬롯은 검정색 작은 점선 테두리(1px dashed #000000)로 표시
- 슬롯 크기는 고정되어 있으며, 이미지는 슬롯 크기에 맞춰 cover 방식으로 표시

- "취소" 시 변경 내용은 저장되지 않는다.

- 회전된 이미지는 출력(PDF/JPEG) 시 편집 모달에서 적용된 회전 상태를 그대로 반영한다.

③ 내용추가(Add Description)

사진별 개별 설명 가능

▷ 텍스트 입력란 동작 규칙
- 슬롯 크기는 절대 고정되어 있어야 함
- 이미지가 업로드된 슬롯에서 "내용추가" 버튼을 클릭하면, 슬롯 내부 하단에 텍스트 입력란(textarea)이 생성됨
- 텍스트 입력란이 생성되면 이미지의 높이가 일부 줄어들고, 줄어든 만큼의 공간에 입력란이 배치됨
- 사용자는 입력란에서 직접 내용을 입력할 수 있으며, 실시간으로 저장됨
- 입력란은 슬롯 내부 하단에 고정되어 있으며, 이미지와 겹치지 않음

▷ 텍스트 스타일
- 폰트 스타일: 기본 Sans-serif
- 줄바꿈 자동 처리
- 위치: 슬롯 내부 하단에 포함되도록 명시 (이미지 바로 하단 고정)
- 최소 높이: 40px

▷ 설명 텍스트 렌더링 및 슬롯 크기 계산 규칙
- 슬롯 전체 크기는 고정되어 있으며 변경되지 않음
- 이미지 영역과 텍스트 입력 영역은 슬롯 내부에서 flex 레이아웃으로 배치됨
- 텍스트 입력란이 활성화되면 이미지 영역은 flex: 1 1 0으로 설정되어 남은 공간을 차지하고, 텍스트 영역은 flex-shrink-0으로 고정 높이를 가짐
- 텍스트 입력란은 자동 줄바꿈되며 슬롯 폭에 맞춰 리플로우됨

9.3.0 다중 페이지 PDF 생성 규칙

▷ 페이지 순서 보장 메커니즘
- 각 페이지는 순차적으로(순서대로) html2canvas로 캡처된다.
- 캡처 순서는 페이지 인덱스(1, 2, 3, ...) 순서를 보장한다.
- 비동기 렌더링 충돌을 방지하기 위해 Promise.all()이 아닌 순차적 await 캡처 방식을 사용한다 (for loop + await).

▷ 캡처 상태 반영 및 안정성 보장
- 다중 페이지 캡처 시 각 페이지는 편집된 최종 슬롯 상태(설명 텍스트 포함)를 기반으로 순서대로 캡처된다.
- 페이지 간 이미지·텍스트의 흐림/누락을 방지하기 위해 각 페이지는 동일한 A4Canvas 상태와 레이아웃 규칙을 공유한다.

▷ 에러 처리 및 페이지 누락 방지
- 특정 페이지 캡처 실패 시, 해당 페이지를 건너뛰고 다음 페이지를 계속 처리한다.
- 모든 페이지 처리 완료 후, 실패한 페이지가 있으면 사용자에게 알림을 표시한다.
- 실패한 페이지 목록을 로그에 기록한다.

▷ 사용자 피드백
- 다중 페이지 출력 시 진행 상황 표시 (예: "3/10 페이지 처리 중...")
- 출력 완료 시 성공/실패 페이지 수 표시

▷ 아웃프레임 크기 일관성 보장
- 각 페이지를 캡처하기 전에 A4Canvas와 OuterFrame의 크기를 동일하게 강제 설정한다.
- 모든 페이지에서 아웃프레임 크기는 210mm × 297mm 기준으로 일관되게 유지된다.
- 캔버스 마진(상·하·좌·우 각 20mm)과 OuterFrame 내부 여백(8mm)은 모든 페이지에서 동일하게 적용된다.
- 이미지 비율이 달라도(가로가 매우 긴 사진, 세로가 매우 긴 사진 등) 아웃프레임 및 캔버스 마진 크기는 변경되지 않는다.
- 각 페이지의 이미지가 서로 다른 비율을 가져도 출력 시 모든 페이지의 아웃프레임 크기는 동일하게 유지된다.

9.3.1 출력 시 UI 요소 제거 규칙

PDF/JPEG 출력 시 다음 UI 요소들은 모두 제거하여 출력되지 않도록 한다:
- 페이지 네비게이션 UI (페이지 N 버튼, 이전/다음 페이지 버튼 등)
- 슬롯 추가 버튼
- 편집/삭제/내용추가 버튼
- 크기 조절 핸들바
- 그 외 편집 전용 표시 요소

**모든 템플릿(2/4/6컷 + 커스텀)에 공통 적용된다.**

출력 시에는 A4Canvas 내부 요소(메타데이터와 이미지 슬롯: 업로드된 이미지 및 해당 설명 텍스트)만 포함된다.

▷ 구현 방법
- PDF/JPEG 출력 시, 편집 UI 요소(슬롯 추가 버튼, 편집/삭제 버튼, 핸들바 등)는 html2canvas가 캡처하기 전에 CSS로 display: none 처리하여 화면에서 숨긴다.
- 출력 완료 후에는 UI를 원래 상태로 복원한다.
- (선택적) 필요 시 출력 전용 캔버스 래퍼를 사용하여 편집 UI가 포함되지 않은 클린 레이어만 캡처하도록 구성할 수 있다.

13.5 인터랙션 및 애니메이션 원칙

로고 hover

scale(1.05~1.08)

Mint 글로우 강도 증가

0.2~0.3s ease-in-out

헤더 메뉴 hover

배경: Neoblue 반투명 → 불투명으로 변화

scale(1.02~1.05)

텍스트 밝기 증가

카드 hover

scale(1.05~1.07)

Neoblue + Mint 글로우

테두리 색 강조

transition: 0.2~0.3s ease-out

버튼 hover

배경 색 조금 더 밝게

미세한 scale(1.03~1.05)

그림자/글로우 소폭 증가

애니메이션 공통 규칙

과하지 않게, 부드럽고 자연스럽게

duration: 0.15~0.3초

easing: ease-out 또는 ease-in-out

크기 변화는 10~15% 이내, 위치 이동은 5px 이내
